# Work Log Format Reference

## File Location

- Active log: `~/.daily-worklog/current.md`
- Archive: `~/.daily-worklog/archive/YYYY-MM-DD.md`

## Entry Format

Each entry follows this pattern:

```
- [HH:MM] <message>
- [HH:MM] [TAG] <message>
```

## Supported Tags

| Tag | Meaning | Effect on `/daily` |
|-----|---------|-------------------|
| `[BLOCKER]` | Something blocking progress | Always shown in Blockers section |
| `[HOY]` | Planned for today (Spanish) | Prioritized in Today section |
| `[TODAY]` | Planned for today (English) | Prioritized in Today section |
| `[AUTO]` | Auto-generated by `/log-auto` | Treated as completed work |

## Example Work Log

```markdown
- [09:15] daily sync con el equipo
- [10:30] terminé refactor del módulo de auth
- [11:00] [BLOCKER] esperando acceso a staging
- [14:00] [HOY] empezar integración con Stripe
- [17:45] [AUTO] Trabajé en feat/stripe-checkout: implementé checkout session y webhook handler (3 commits)
```

## Archival Rules

- Entries older than 3 days are moved to archive
- Archive filename uses the date of the oldest entry in the batch
- The archive directory is created automatically
