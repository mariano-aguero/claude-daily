# Work Log Format Reference

## File Location

- Active log: `~/.daily-worklog/current.md`
- Auto-archive (via `/log`): `~/.daily-worklog/archive/YYYY-MM-DD.md`
- Full archive (via `/log-clear`): `~/.daily-worklog/archive/YYYY-MM-DD-HH-MM.md`

## Entry Format

Each entry includes full date + time for reliable temporal filtering:

```
- [YYYY-MM-DD HH:MM] <message>
- [YYYY-MM-DD HH:MM] [TAG] <message>
```

## Supported Tags

| Tag | Meaning | Effect on `/daily` |
|-----|---------|-------------------|
| `[BLOCKER]` | Something blocking progress | Always shown in Blockers section |
| `[HOY]` | Planned for today (Spanish) | Prioritized in Today section |
| `[TODAY]` | Planned for today (English) | Prioritized in Today section |
| `[AUTO]` | Auto-generated by `/log-auto` | Treated as completed work |
| `[AUTO-STOP]` | Auto-generated by session stop hook | Treated as completed work summary |
| `[COMPACT]` | Context compaction marker (internal) | Ignored by `/daily` |

## Example Work Log

```markdown
- [2026-03-08 09:15] daily sync con el equipo
- [2026-03-08 10:30] terminé refactor del módulo de auth
- [2026-03-08 11:00] [BLOCKER] esperando acceso a staging
- [2026-03-08 14:00] [HOY] empezar integración con Stripe
- [2026-03-08 17:45] [AUTO] Trabajé en feat/stripe-checkout: implementé checkout session y webhook handler (3 commits)
```

## Archival Rules

- Entries older than 3 days are automatically moved to archive when `/log` is used
- Only entries with the `[YYYY-MM-DD` date prefix are subject to auto-archival
- Archive filename uses the current date (not the entry date)
- Run `/log-clear` manually to archive everything and start fresh (recommended on Monday mornings)
